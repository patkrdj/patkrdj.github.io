## 배경

Java는 객체지향 언어지만 RDB(Relational Database)는 객체를 사용하지 않기 때문에 패러다임 불일치가 발생한다. 그렇기 떄문에 객체와 테이블을 연결해주는 기술을 ORM(Object-Relational Mapping)이라고 부르고 JPA(Java Persistence API)는 ORM의 Java 표준 스펙이다. JPA를 사용하면 단순하고 반복적인 CRUD query를 짤 필요가 없게 된다.

※ MongoDB와 같은 json 방식으로 저장하는 NoSQL은 JPA가 필요없다. 하지만 복잡한 트랜잭션은 RDB가 사용하는 것이 유리하다.

## 영속성(Persistence)

영속성이란 데이터가 프로그램을 종료해도 물리적 디스크에 영구적으로 저장되는 성질이다. JPA 환경에서는 특히 객체의 성질을 데이터베이스와 동기화시켜서 관리하는 것을 뜻한다. 영속성 컨텍스트는 JPA의 핵심으로 JPA가 Entity를 관리하는 가상의 공간이다.

### Entity의 4가지 상태

1. 비영속(transient) : 순수한 객체의 상태
2. 영속(managed) : 영속성 컨텍스트가 관리하는 상태, JPA가 객체의 상태를 확인하고 DB와 동기화
3. 준영속(detached) : 영속성 컨텍스트와 분리된 상태, 더 이상 JPA의 관리를 받지 않음
4. 삭제(removed) : 삭제를 위한 마크된 상태, 트랜잭션이 완료될 때 DB에서 실제로 삭제됨

## Spring Boot 코드

**Hibernate** : Spring Boot에서 ****JPA 규격을 실제로 변환하는 소프트웨어(Entity → SQL query)

### Reflection 기술

Reflection이란 runtime 도중에 자기 자신의 구조(클래스, 메소드, 필드)를 뜯어보고 조작하는 기술이다. 이 기술이 필요한 이유는 Hibernate는 자신의 코드 안에서 객체를 동적으로 다뤄야 하기 때문이다. 만약 정적으로 객체를 다뤄야 한다면 Hibernate의 코드를 열어서 else if로 자신이 추가한 객체를 일일이 넣어줘야 한다. 그리고 다른 이유로는 private field 값을 우회적으로 접근하기 위해서이다.

※ Encapsulation 원칙이 깨지는 것는 디자인 상의 anti-pattern이지만 우리가 작성하는 비즈니스 코드가 깔끔한 객체지향의 형태를 유지하기 위해서 사용한다.

### Entity

DB에 어떤 식으로 mapping될지 결정하는 클래스이다. 또한 어떤 column이 메모리 상에서 관리할지 결정할 수 있다.

```java
@Entity // JPA(Hibernate)에게 요청
@Getter
// JPA(Hibernate)의 Reflection 기술을 사용하기 위함
@NoArgsConstructor(access = AccessLevel.PROTECTED)
public class Book {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
		
    @Column(length = 20)
    private String isbn;

    @Column(nullable = false, length = 100)
    private String title;

    @Column(length = 50)
    private String author;

    // 생성자에서 Id 값을 결정하면 안되므로 lombok 생성자를 사용하지 않음
    @Builder
    public Book(String isbn, String title, String author) {
        this.isbn = isbn;
        this.title = title;
        this.author = author;
    }
}
```

### DTO

DTO(Data Transfer Object)는 Entity를 직접적으로 접근하는 것을 막고 필요한 데이터만을 사용하기 위한 클래스이다. Entity와는 1:N의 관계를 가지며 클라이언트의 요구에 맞춰서 파생된 클래스이다.

```java
public class BookDto {
    @Getter
    // {Jackson 변환기}가 JSON 데이터를 Java Object 로 변환할 때 필요
    @NoArgsConstructor
    //여기서 `static`은 외부 클래스가 생성되지 않아도 생성될 수 있음을 의미
    public static class CreateRequest {
        private String title;
        private String author;
    }

    @Getter
    @NoArgsConstructor
    public static class SearchResponse {
        private List<SearchItem> items;
    }

    @Getter
    @NoArgsConstructor
    public static class SearchItem {
        private String title;
        private String image;
        private String author;
        private String publisher;
        private String pubdate;
        private String isbn;
        private String description;
    }
}
```

※ Jackson은 json 형태의 데이터와 java 객체로 변환하는 역할을 하며, HTTP 통신에 사용하는 json과 DTO를 변환한다.

### Repository

Repository는 Hibernate를 사용하여 Entity를 SQL query로 변환하는 인터페이스이며, JSQL query를 새로 작성하거나 override할 수 있다.

```java
public interface ReviewRepository extends JpaRepository<Review, Long> {
    // `JPQL`을 따르기 때문에 r 이라는 객체를 그대로 `SELECT` 안에 넣어도 된다.
    @Query("SELECT r FROM Review r JOIN FETCH r.user WHERE r.book.isbn = :isbn")
    List<Review> findAllByBookIsbn(@Param("isbn") String isbn);
    
    // `existsBy...`을 보고 알아서 {JPA}가 알아서 `SELECT COUNT(...) > 0` 이러한 쿼리를 작성
		// 따라서 `@query`를 추가하지 않아도 됨.
		boolean existsByContent(String content);
}
```

Spring은 메모리 상에서 인터페이스를 생성하는 Dynamic Proxy를 생성하게 된다. 내부적으로 `SimpleJpaRepository` 로 클래스와 연결되며 Dynamic Proxy가 요청을 받으면 Hibernate에게 알리는 방식으로 동작한다.

### Service

Entity, DTO, Repository를 조립하여 비즈니스 로직을 구현하는 클래스이다. 여기서 여러 번의 DB 접근을 묶어서 Transaction으로 관리할 수 있도록 한다.

```java
@Service    // DI와 Singleton 패턴을 사용하기 위해
@RequiredArgsConstructor
public class BookService {
    public final BookRepository bookRepository;

    @Transactional  // Hibernate에게 요청, All or Nothing 지키기 위함
    public String createBook(BookDto.CreateRequest request) {
        Book newBook = Book.builder()
                .title(request.getTitle())
                .author(request.getAuthor())
                .build();

        return bookRepository.save(newBook).getIsbn();
    }
}
```
